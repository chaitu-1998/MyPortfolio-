First of all, OOP is the most frequent and widely used programming paradigm which allows us to code by thinking like a real-world objects and their relationships to another. The majority of the popular programming languages using OOP such as, Java, C++, Python and more…

Object-Oriented is a super set of Imperative Programming. So, what is Imperative Programming? It is one of the main paradigms which employs statements to modify the state of a program.

A statement is a command that we give to the system can be a variable or a method, for example:

// That can be a variable

int isAge = 17;
boolean isStudent = true;
String isName = "Ahmet Fatih";


// Or a method...
System.out.println("It is a method!");

We have assigned the name variable as “Ahmet Fatih” with our age 17. We also created a isStudent boolean as true. So, all of these are statements assigned to the computer that; Ahmet Fatih is a 17 years old student.

I wish we have understood the logic of Imperative Paradigm. Now, we can start to explain the pure Object-Oriented Programming. As we said before, this model uses “user-defined objects” which contains data (information) about themselves. Each object is located in classes, so we can use them whenever it needed. To define what is a class, it is a blueprint for creating objects. Finally, this case is called an “Inheritance”.
Let’s practice by creating a class which gives information about me:

// We put the statements in the class:

class A {
      int isAge = 17;
      boolean isStudent = true;
      String isName = "Ahmet Fatih";
}

We can also represent our code like this:

As we can see, Ahmet Fatih became a class named A. He is now known as “A” by the computer. That means, he is represented by this class.

We can also use inheritance with methods, as we have seen before. For example:

// We create our methodClass to assign our method...

class MethodClass{
          isMethod() {
    System.out.println("It is a method!");
  }

// Now we use it in main class!

 public static void main(String[] args) {
    MethodClass();
  }
}

This code shows also briefly our “Each object is located in classes, and we can use them whenever it needed” definition.

OOP also allows to put important personal information in our class by restricting its accessibility. To do it, we use the “Variable and Method Hiding”. For example:

class A {
  public int isAge = 17;
  public boolean isStudent = true;
  public String isName = "Ahmet Fatih";

// We're adding our credit card information.

private int cCardNum = 4716707011182869;
  private int expDateM = 8;
  private int expDateY = 2026;
  private int cvcNum = 631;
}

We added the credit card information of Class A with the “private” keyword. So they will not be public accessible in contrary to isName, isStudent and isAge statements.

Then, this paradigm contains also the “Polymorphism” which works with the “Inheritance”. It means “many forms”, and it occurs when we have many classes that are related to each other by inheritance. So, it is related with “Inheritance” which we explained just now. Polymorphism uses the methods to perform different tasks. This allows us to perform a single action in different ways.

To explain clearly, we will add a classmate named “Jonathan” with me. Before adding him, we will create a superclass of “Students” which we can add also as many students as we want in it. Then, we will assign us as a subclass of “Students” by using the extends keyword. So, different students will give their own names by calling heSays() method on both of them.

Here it is:

// Let's create the superclass:

class Students {
  public void heSays() {
    System.out.println("We are students and our names are different!");
  }
}
// Let's create the subclasses for just me, I will keep the name A for me:
class A extends Students {
  public void heSays() {
    System.out.println("My name is Ahmet Fatih!");
  }
}
// And I will create the subclass of Jonathan as J:
class J extends Students {
  public void heSays() {
    System.out.println("My name is Jonathan!");
  }
}

And now we call them in the main class:

class Main {
  public static void main(String[] args) {

Students allStudents = new Students();  // We created the students object
    Students StudentA = new A();  // We created me
    Students StudentJ= new J();  // We created Jonathan

// Let's call our method:ş
    allStudents.heSays();
    StudentA.heSays();
    StudentJ.heSays();
  }
}

The output will be:

We are students and our names are different!
My name is Ahmet Fatih!
My name is Jonathan!

So, we now almost understood the Object-Oriented Programming principles. Such as, the main Imperative Programming paradigm. What are the Objects and Classes. How can we use them with the Inheritance, Hiding and Polymorphism concepts.
